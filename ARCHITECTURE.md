## 🔄 دورة حياة الطلب

### 1. استقبال الرسالة

```
on_link()
├─ فحص صحة الرابط
├─ تسجيل المستخدم
├─ إنشاء رسالة حالة مؤقتة
└─ إطلاق do_download() بالتوازي
```

### 2. معالجة التحميل

```
do_download()
├─ توليد معرف فريد
├─ إنشاء hook للتحديثات
│  ├─ حساب السرعة
│  ├─ حساب الوقت المتبقي
│  └─ تحديث الرسالة
├─ التحقق من نوع المحتوى
│  ├─ YouTube → محاولة Invidious
│  └─ الآخرون → direct yt-dlp
├─ تحميل الملف
│  ├─ معالجة الأخطاء
│  ├─ إعادة المحاولة
│  └─ التحقق من الملف
└─ المتابعة: حسب حجم الملف
```

### 3. بعد التحميل

```
Post-Download:
├─ حجم < 48MB
│  ├─ إرسال مباشرة
│  └─ حذف المؤقت
└─ حجم >= 48MB
   ├─ تقسيم FFmpeg
   ├─ إرسال الأجزاء
   └─ حذف الأجزاء
```

## 🗄️ هيكل قاعدة البيانات

### جدول المستخدمين

```sql
CREATE TABLE users(
    uid         INTEGER PRIMARY KEY,
    username    TEXT,
    fname       TEXT,
    seen        TEXT,
    active      TEXT,
    cnt         INTEGER DEFAULT 0
);
```

**الفهارس:**
- `uid` (Primary Key): معرف Telegram الفريد

**العلاقات:**
- علاقة one-to-many مع جدول downloads

### جدول التحميلات

```sql
CREATE TABLE downloads(
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    uid         INTEGER,
    url         TEXT,
    title       TEXT,
    quality     TEXT,
    size        INTEGER,
    speed       REAL,
    ts          TEXT
);
```

**الفهارس:**
- `id` (Primary Key): معرف التحميلة الفريد
- `uid` (Foreign Key): معرف المستخدم

### تحسينات الأداء

```python
db.execute("PRAGMA journal_mode=WAL")      # Write-Ahead Logging
db.execute("PRAGMA synchronous=NORMAL")    # توازن بين الأمان والسرعة
```

## 🚀 تسلسل التحميل التفصيلي

### للفيديو العادي:

```
1. استقبال الرابط
   └─ رابط YouTube؟
      ├─ نعم → محاولة Invidious
      └─ لا → yt-dlp مباشرة

2. تحميل الملف
   ├─ بدء yt-dlp
   ├─ تحديث التقدم كل ثانية
   │  ├─ سرعة = (البايتات الجديدة / الدلتا الزمن) / 1024
   │  ├─ ETA = (الباقي) / السرعة
   │  └─ تحديث الرسالة
   └─ انتظار الانتهاء

3. معالجة بعد التحميل
   ├─ حساب الحجم
   ├─ حساب السرعة الكلية
   ├─ تسجيل في DB
   └─ الإرسال
```

### مع التقسيم:

```
1-2. (نفس الخطوات أعلاه)

3. التقسيم
   ├─ حساب المدة
   ├─ حساب الأجزاء المطلوبة
   ├─ تشغيل FFmpeg
   └─ انتظار الانتهاء

4. الإرسال
   ├─ لكل جزء:
   │  ├─ فتح الملف
   │  ├─ إرسال إلى Telegram
   │  ├─ إغلاق الملف
   │  └─ حذف الملف
   └─ تسجيل النجاح
```

## 📊 معالجة الأخطاء

### شجرة قرار معالجة الأخطاء

```
استثناء؟
├─ Sign in / Bot verification
│  └─ محاولة Invidious → ناجح؟
│     ├─ نعم → استمرار
│     └─ لا → خطأ نهائي
├─ Requested format not available
│  └─ إعادة محاولة بصيغة "best"
├─ Network error
│  └─ إعادة محاولة (yt-dlp يفعلها تلقائياً)
└─ Other errors
   └─ إرسال رسالة خطأ للمستخدم
```

### آليات إعادة المحاولة

```
yt-dlp options:
├─ retries: 10              # إعادة محاولة الملف
├─ fragment_retries: 10     # إعادة محاولة الأجزاء
├─ continuedl: True         # متابعة التحميل
└─ socket_timeout: 60       # انتظر 60 ثانية
```

```

3. معالجة الأخطاء
   └─ لا تعرض معلومات حساسة

4. تنظيف الملفات
```

## 📈 قابلية التوسع

### نقاط التوسع المستقبلية:

1. **دعم المزيد من المنصات**
   - أضف معالجات خاصة في `on_link()`

2. **جودات متقدمة**
   - أضفها إلى قاموس `QUALITY`

3. **الإحصائيات المتقدمة**
   - أضف استعلامات جديدة إلى `on_admin()`

4. **المراقبة والتسجيل**
   - أضف logging متقدم

5. **الترجمة**
   - استخدم حزمة i18n
